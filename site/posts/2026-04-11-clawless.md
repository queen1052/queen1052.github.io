## ClawLess란 무엇인가?

AI 에이전트를 실행하려면 보통 서버가 필요합니다. API 서버, 포트 개방, 인프라 설정... 이 모든 것이 진입 장벽이 됩니다. **ClawLess**는 이 문제를 근본적으로 해결합니다. WebContainers(WASM) 기술을 이용해 브라우저 안에서 완전한 Node.js 런타임을 실행하고, 그 위에서 AI 에이전트가 동작하도록 합니다. 서버가 전혀 필요 없습니다.

클라우드 없이 브라우저 탭 하나로 AI 에이전트를 완전히 실행할 수 있다는 것, 이것이 ClawLess의 핵심 가치입니다. npm 패키지 340만 개 이상에 접근할 수 있고, 파일 시스템도 가상으로 작동하며, 모든 데이터는 사용자의 브라우저 안에만 머뭅니다.

## 주요 기능 한눈에 보기

ClawLess는 단순한 에이전트 실행 환경이 아닙니다. 개발자가 필요로 하는 모든 도구가 통합된 풀스택 에이전트 플랫폼입니다.

- **WebContainer 샌드박스**: OS 수준의 격리, WASM 기반으로 호스트 시스템에 접근 불가
- **Monaco 에디터**: VS Code의 핵심 에디터, 멀티 파일 탭 지원
- **xterm.js 터미널**: 완전한 PTY 지원의 실제 터미널 세션
- **GitHub 연동**: API를 통한 저장소 클론 및 푸시
- **YAML 정책 엔진**: 에이전트 행동을 YAML로 선언적으로 제어
- **완전한 감사 로그**: 프로세스, 파일, 네트워크, git 이벤트 전체 기록
- **플러그인 시스템**: 라이프사이클 훅으로 확장 가능
- **멀티 AI 지원**: Anthropic(Claude), OpenAI(GPT), Google(Gemini) 지원

## WebContainer 샌드박스 심층 탐구

ClawLess의 핵심은 WebContainers API입니다. 이 기술은 브라우저 내에서 실제 OS 환경을 에뮬레이션합니다.

에이전트가 파일을 생성하면? 가상 파일 시스템에 저장됩니다. 에이전트가 npm 패키지를 설치하면? 실제로 설치되고 실행됩니다. 에이전트가 웹서버를 시작하면? 브라우저 내에서 동작합니다. 이 모든 것이 WASM 샌드박스 안에서 이루어지므로 호스트 시스템에는 아무런 영향을 주지 않습니다.

실제 예시로 ClawLess 데모에서는 에이전트가 `pptxgenjs`를 설치하고 9슬라이드짜리 파워포인트 프레젠테이션을 생성합니다. 모든 것이 서버 없이 브라우저 안에서 완료됩니다.

## YAML 정책 엔진: 에이전트 행동 제어

에이전트가 무엇을 할 수 있고 없는지를 YAML로 정의합니다. 파일 접근 규칙, 허용 프로세스, 포트 바인딩, 최대 파일 크기, 최대 프로세스 수, 타임아웃 등을 선언적으로 제어합니다.

정책은 컨테이너 레벨에서 강제 적용되므로 에이전트가 우회할 수 없습니다. 정책을 재적용해도 컨테이너를 재시작할 필요가 없으며, 인터페이스에서 실시간으로 적용/초기화가 가능합니다.

감사 로그는 모든 액션을 추적합니다: 프로세스 라이프사이클, 파일 I/O, 네트워크 요청/응답, 환경 설정, 정책 적용 여부. API 키 같은 민감한 헤더는 자동으로 마스킹됩니다.

## 기술 스택 및 아키텍처

ClawLess의 주요 구성 요소는 다음과 같습니다:

| 컴포넌트 | 역할 |
|---------|------|
| ClawContainer SDK | 소비자를 위한 단일 진입점 |
| ContainerManager | WebContainer 오케스트레이션 및 라이프사이클 |
| PolicyEngine | 파일/프로세스/네트워크 규칙 강제 |
| AuditLog | 컨테이너 내 모든 액션 완전 기록 |
| GitService | GitHub API 연동(클론, 커밋, 푸시) |
| PluginManager | 라이프사이클 훅으로 동작 확장 |
| UIManager | Monaco 에디터, xterm.js 터미널, 탭 관리 |

기술 스택: **Vite + TypeScript** (빠른 빌드, 타입 안전), **WebContainer API** (브라우저 네이티브 OS 환경), **xterm.js** (풀 피처드 터미널 에뮬레이터), **Monaco Editor** (VS Code의 에디터 엔진)

## 설치 및 사용법

### 로컬 실행

```bash
git clone https://github.com/open-gitagent/clawless.git
cd clawless
npm install
npm run dev
```

### npm 패키지로 설치

```bash
npm install clawcontainer
```

### SDK 사용 예시

```javascript
import { ClawContainer } from 'clawcontainer';

const cc = new ClawContainer('#app', {
  template: 'gitclaw',
  env: { ANTHROPIC_API_KEY: 'sk-...' }
});

await cc.start();
cc.on('ready', () => console.log('Container ready!'));
```

환경 변수로 API 키를 주입하면 됩니다. 세션 상태는 `localStorage`의 `clawchef_` 접두사 하에 저장되어 페이지 새로고침 후에도 유지됩니다. 호스팅 버전은 [play.clawless.io](https://play.clawless.io/)에서 바로 체험 가능합니다.

## 마치며

ClawLess는 AI 에이전트 개발의 진입 장벽을 획기적으로 낮춥니다. 서버 없이, 인프라 설정 없이, 브라우저 하나로 완전한 에이전트 개발 환경을 제공합니다. YAML 정책 엔진과 완전한 감사 로그로 보안과 관찰 가능성도 챙겼습니다. GitAgent 표준을 따르는 플러그인 가능한 SDK와 템플릿 시스템으로 확장도 용이합니다. 256 스타를 기록하는 이 신생 프로젝트가 서버리스 AI 에이전트 런타임의 새로운 표준이 될지 주목할 만합니다.

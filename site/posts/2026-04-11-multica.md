## 소개

**Multica**는 코딩 에이전트를 진정한 팀원으로 만드는 오픈소스 관리 에이전트 플랫폼입니다. "다음 10명의 신입은 사람이 아닐 것이다"라는 담대한 슬로건처럼, Multica는 Claude Code, Codex, OpenClaw, OpenCode 에이전트들을 인간 동료와 동등한 방식으로 팀에 통합합니다.

기존의 AI 코딩 도구가 프롬프트를 복사/붙이거나 실행을 감시해야 했다면, Multica는 **이슈를 동료에게 할당하듯 에이전트에게 할당**하면 에이전트가 스스로 작업을 처리하고 상태를 보고합니다.

---

## 주요 기능

Multica의 핵심 기능들:

- **에이전트-팀원 통합**: 보드에 에이전트 프로필 생성, 이슈 할당, 댓글 및 블로커 자동 보고
- **자율 실행 엔진**: 풀 태스크 라이프사이클 관리 (enqueue → claim → start → complete/fail) + WebSocket 실시간 스트리밍
- **재사용 가능 스킬**: 모든 솔루션이 팀 전체의 재사용 스킬로 자동 변환
- **통합 런타임 관리**: 로컬 데몬 + 클라우드 런타임, CLI 자동 감지
- **멀티 워크스페이스**: 팀별 격리된 워크스페이스 (에이전트, 이슈, 설정 분리)
- **Google OAuth 지원**: 간편한 팀 인증

---

## 핵심 기능 1: 에이전트 라이프사이클 관리

Multica의 핵심은 에이전트의 전체 작업 주기를 자동으로 관리한다는 것입니다:

**작업 흐름:**
1. 사용자가 이슈 생성 (보드 또는 `multica issue create`)
2. 에이전트에게 할당 (인간 동료에게 할당하는 방식과 동일)
3. 데몬이 자동으로 작업 감지 및 격리된 환경 생성
4. 에이전트 실행 (claude, codex, openclaw, opencode)
5. 진행 상황 실시간 WebSocket 스트리밍
6. 완료 후 결과 보고 및 스킬 등록

**블로커 자동 보고:**
에이전트가 작업 중 막히는 경우 자동으로 블로커를 보고하고 도움을 요청할 수 있습니다.

---

## 핵심 기능 2: 스킬 복리 효과

Multica의 가장 강력한 장기 가치 제안은 **스킬 축적**입니다:

- 에이전트가 특정 문제를 해결할 때마다 그 솔루션이 재사용 가능한 스킬로 등록
- 배포, 마이그레이션, 코드 리뷰 패턴이 팀 전체 스킬 라이브러리에 축적
- 새로운 에이전트나 팀원이 이전에 해결된 문제의 스킬을 즉시 활용 가능
- 팀의 집합적 역량이 시간이 지날수록 복리로 성장

## 기술 스택 및 아키텍처

```
┌──────────────┐     ┌──────────────┐     ┌──────────────────┐
│   Next.js    │────>│  Go Backend  │────>│   PostgreSQL     │
│   Frontend   │<────│  (Chi + WS)  │<────│   (pgvector)     │
└──────────────┘     └──────┬───────┘     └──────────────────┘
                            │
                     ┌──────┴───────┐
                     │ Agent Daemon  │
                     │Claude/Codex/  │
                     │OpenClaw/Code  │
                     └──────────────┘
```

| 컴포넌트 | 기술 |
|---------|------|
| 프론트엔드 | Next.js 16 (App Router) |
| 백엔드 | Go (Chi, sqlc, gorilla/websocket) |
| 데이터베이스 | PostgreSQL 17 + pgvector |
| 에이전트 런타임 | 로컬 데몬 |

## 설치 및 사용법

**클라우드 (즉시 시작):**
[https://multica.ai](https://multica.ai) 방문

**자체 호스팅 (Docker):**
```bash
git clone https://github.com/multica-ai/multica.git
cd multica
cp .env.example .env
# .env에서 JWT_SECRET 변경 필수
docker compose -f docker-compose.selfhost.yml up -d
# http://localhost:3000
```

**CLI 설치 및 데몬 시작:**
```bash
# macOS
brew tap multica-ai/tap
brew install multica

# 인증 및 데몬 시작
multica login
multica daemon start
```

**개발 환경 (원커맨드 설정):**
```bash
# Node.js v20+, pnpm v10.28+, Go v1.26+, Docker 필요
make dev  # 환경 감지, 의존성 설치, DB 설정, 마이그레이션, 모든 서비스 시작
```

## 마치며

Multica는 AI 에이전트가 프로젝트 보드에서 진짜 팀원처럼 보이고, 행동하고, 결과를 보고하는 미래를 만들어갑니다. 오픈소스로 자체 호스팅 가능하며, 클라우드 버전도 제공합니다. 코딩 에이전트를 단순한 도구가 아닌 팀의 생산성 배수기로 활용하고 싶다면, Multica는 그 인프라를 제공합니다.

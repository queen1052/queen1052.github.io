## 소개

Trendshift는 Julian Li가 만든 **오픈소스 트렌드 탐색 플랫폼**입니다. [trendshift.io](https://trendshift.io)에서 운영되며, GitHub의 기본 트렌딩 기능을 넘어서 **AI 기반 자동 분류, 일별 인기 저장소, 주제별 탐색**을 제공합니다. 개발자들이 매일 최신 오픈소스 트렌드를 효율적으로 파악할 수 있는 큐레이션 도구입니다.

GitHub의 공식 트렌딩 페이지는 기본적인 기능만 제공합니다: 일간/주간/월간 Top 저장소를 언어별로 필터링하는 것이 전부입니다. Trendshift는 여기에 더해 **AI가 저장소를 자동으로 토픽별로 분류**하고, 주제별 트렌드 흐름을 추적하며, 특정 기술 스택(예: "React 기반 AI 에이전트 UI")처럼 복합적인 기준으로 저장소를 탐색할 수 있게 합니다.

오픈소스 커뮤니티에서 주목할 새로운 프로젝트를 발견하거나, AI/ML/Web 같은 분야의 트렌드 흐름을 모니터링하거나, 특정 기술의 채택 속도를 파악하는 데 활용됩니다.

## 주요 기능

- **AI 자동 분류**: 각 저장소의 README, 설명, 코드를 AI가 분석해 토픽과 카테고리를 자동으로 분류합니다. 태그가 없는 저장소도 올바른 카테고리에 배치됩니다.
- **일별 트렌딩 목록**: 매일 업데이트되는 인기 저장소 목록. 빠르게 상승하는 저장소를 일찍 발견해 흐름을 파악합니다.
- **주제별 탐색**: AI 에이전트, LLM, WebAssembly, Rust 등 특정 주제의 인기 저장소를 탐색합니다. 관심 기술 분야의 최신 동향을 집중적으로 확인합니다.
- **트렌드 히스토리**: 특정 저장소나 주제의 인기 추이를 시간축으로 확인합니다. 어떤 기술이 상승하고 어떤 기술이 하락하는지 데이터로 파악합니다.
- **멀티 플랫폼 트렌드**: GitHub를 중심으로 하되 관련 커뮤니티(Hacker News, Reddit r/programming, Product Hunt)의 반응도 통합해 종합적인 인기도를 측정합니다.
- **개인화 피드**: 관심 기술 스택과 주제를 설정하면 맞춤화된 트렌딩 피드가 제공됩니다.

### GitHub 트렌딩 vs Trendshift

| 기능 | GitHub 트렌딩 | Trendshift |
|------|--------------|-----------|
| 업데이트 | 일간/주간/월간 | 매일 |
| 분류 | 프로그래밍 언어 | AI 자동 토픽 분류 |
| 검색 | 없음 | 주제별 탐색 |
| 히스토리 | 없음 | 트렌드 추이 |
| 크로스 플랫폼 | 없음 | HN, Reddit 통합 |
| 개인화 | 없음 | 맞춤 피드 |

Trendshift의 가장 큰 강점은 **AI 자동 분류**입니다. GitHub에서 "AI"를 검색하면 수천 개의 결과가 나오지만 품질이 천차만별입니다. Trendshift는 실제로 최근 주목받는 고품질 AI 관련 저장소만 분류해 보여줍니다.

## 기술 스택 / 아키텍처

| 항목 | 내용 |

| 데이터 소스 | GitHub API, HN, Reddit |
| 분류 | AI 기반 자동 토픽 분류 |
| 개발자 | Julian Li |
| 업데이트 주기 | 매일 |
| 접근 방식 | 웹 앱 + API |

## 사용법

```bash
# API 접근 (있는 경우)
curl "https://api.trendshift.io/v1/trending?period=daily&topic=ai-agents" \
  -H "Authorization: Bearer YOUR_API_KEY"

# RSS 피드 구독
https://trendshift.io/feed/ai-agents.rss
https://trendshift.io/feed/llm.rss

# 북마클릿으로 현재 저장소 트렌드 확인
javascript:(function(){
  window.open('https://trendshift.io/repo/' + window.location.pathname.split('/').slice(-2).join('/'));
})()
```

웹 앱:
1. [trendshift.io](https://trendshift.io) 방문
2. 관심 주제 선택 또는 검색
3. 일별 인기 저장소 확인
4. 계정 생성 후 개인화 피드 설정

## 활용 사례 / 사용 시나리오

1. **개발자 일과 시작 루틴**: 매일 아침 Trendshift로 AI 에이전트, TypeScript, Rust 등 관심 주제의 신규 인기 저장소를 5분 안에 파악합니다. 기술 트렌드를 놓치지 않고 학습 방향을 최신 상태로 유지합니다.

2. **투자/비즈니스 의사결정**: 특정 기술 영역의 오픈소스 활동 증가는 상용 제품 수요 증가의 선행 지표입니다. Trendshift의 트렌드 히스토리로 기술 방향을 데이터 기반으로 평가합니다.

3. **콘텐츠 아이디어 발굴**: 기술 블로그 운영자나 유튜버가 Trendshift에서 빠르게 성장하는 저장소를 발견하고, 여론이 형성되기 전에 먼저 콘텐츠를 준비합니다.

## 결론

Trendshift는 GitHub 트렌딩의 제한적인 기능을 넘어서 AI 분류와 주제별 탐색으로 오픈소스 트렌드 발견을 더 효율적으로 만들어줍니다. 일별 업데이트와 트렌드 히스토리로 단순한 인기 목록이 아닌 기술 트렌드의 흐름을 파악할 수 있습니다. 최신 기술 동향에 앞서가고 싶은 개발자라면 매일 방문할 가치가 있는 플랫폼입니다.

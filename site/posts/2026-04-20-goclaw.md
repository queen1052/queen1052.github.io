## 소개

GoClaw: Go 기반 멀티테넌트 AI 에이전트 플랫폼는 보안과 멀티테넌시를 강조한 Go 기반 AI 에이전트 게이트웨이입니다.라는 설명이 잘 어울리는 프로젝트입니다. 이 저장소가 흥미로운 이유는 단순히 기능 목록을 늘어놓는 데서 끝나지 않고, Go로 재구성한 multi-tenant AI agent gateway를 제공하는 것 같은 실제 문제를 비교적 분명한 방식으로 다루기 때문입니다.

README를 기준으로 보면 이 프로젝트는 Go로 재구성한 multi-tenant AI agent gateway를 제공하는 것라는 목표를 위해 필요한 구성 요소를 직접적으로 드러냅니다. 사용자는 복잡한 시행착오 없이 빠르게 결과를 확인할 수 있고, 팀 입장에서는 반복 가능한 흐름을 바탕으로 도입 여부를 판단하기 쉽습니다. 오픈소스 프로젝트를 고를 때 중요한 것은 화려한 데모보다도 “내 작업에 바로 연결할 수 있는가”인데, 이 저장소는 그 점에서 꽤 선명한 인상을 줍니다.

## 주요 기능

- 20개 이상의 LLM provider와 7개 메시징 채널을 통합해 대규모 운영을 겨냥합니다.
- Knowledge Vault와 3-tier memory로 문서와 세션 기억을 체계적으로 축적합니다.
- Go 단일 바이너리 기반으로 배포가 단순하고 스타트업이 빠르게 self-hosted 운영을 시작하기 좋습니다.
- 5-layer security, RBAC, encrypted API key 저장 등 운영 보안에 신경 쓴 설계가 돋보입니다.
- Lite 데스크톱 에디션까지 제공해 로컬 테스트부터 서버 운영까지 연결 가능합니다.

### 핵심 포인트 1

이 프로젝트의 첫 번째 강점은 20개 이상의 LLM provider와 7개 메시징 채널을 통합해 대규모 운영을 겨냥합니다.라는 점입니다. 단순한 샘플 수준이 아니라 실제 문제를 어떤 데이터 흐름과 도구 체계로 풀어낼지 보여 주기 때문에, 처음 접하는 사람도 도입 방향을 잡기 쉽습니다. 또한 Knowledge Vault와 3-tier memory로 문서와 세션 기억을 체계적으로 축적합니다. 같은 요소는 실사용 단계에서의 마찰을 줄여 주며, 기존 작업 환경과 자연스럽게 연결될 가능성을 높여 줍니다.

### 핵심 포인트 2

아키텍처 관점에서도 8단계 에이전트 파이프라인, 3계층 메모리, PostgreSQL 기반 멀티테넌트 구조라는 점이 분명합니다. 이런 구조는 유지보수성과 확장성에 직접 영향을 주며, 향후 새로운 모델·도구·워크플로우를 붙일 때도 부담을 줄여 줍니다. 특히 단일 바이너리와 강한 보안 모델로 대규모 운영을 노린다는 점은 비슷한 오픈소스와 비교할 때 이 프로젝트를 다시 보게 만드는 차별점입니다.

## 설치 / 사용법

아래는 README 기준으로 시작할 때 참고하기 좋은 대표 명령들입니다.

```bash
git clone -b main https://github.com/nextlevelbuilder/goclaw.git && cd goclaw
cd goclaw
make build
./goclaw onboard
```

## 소개: 금융 리서치를 위해 설계된 자율 에이전트

"Think Claude Code, but built specifically for financial research." — Dexter README의 설명이다.

**Dexter**는 virattt(Virat Thatavarthi)가 개발한 자율 금융 리서치 에이전트다. GitHub **21.2k stars**, TypeScript 99.4%, MIT 라이선스로 공개되어 있다. 22명의 기여자, 17개 릴리즈를 기록한 빠르게 성장하는 프로젝트다.

Dexter의 작동 방식은 단순하다: 복잡한 금융 질문(예: "애플의 지난 5년 FCF 성장률과 현재 밸류에이션을 분석해줘")을 받으면 → 단계별 리서치 계획을 수립하고 → 실시간 시장 데이터를 도구로 수집하며 → 자체 검증 후 데이터 기반 답변을 제공한다.

## 주요 기능: 자율적인 금융 분석 파이프라인

**지능형 태스크 플래닝**: 복잡한 쿼리를 구조화된 리서치 단계로 자동 분해한다. "NVIDIA와 AMD의 실적을 비교해줘"라고 물으면, AI가 어떤 지표를 수집하고 어떤 순서로 분석할지 계획을 세운다.

**자율 실행**: 올바른 도구를 선택하고 실행해 금융 데이터를 수집한다. Income Statement(손익계산서), Balance Sheet(재무상태표), Cash Flow Statement(현금흐름표)에 실시간으로 접근한다.

**자기 검증(Self-Validation)**: 수집한 결과를 스스로 확인하고, 불완전하거나 불충분하면 반복해서 보완한다. 태스크가 완료될 때까지 계속 시도한다는 명확한 종료 조건이 있다.

**안전 기능**: 루프 감지와 단계 제한(Step Limit)으로 무한 실행을 방지한다. 비용 관리에 중요한 기능이다.

**실시간 금융 데이터**: Financial Datasets API(기관급 시장 데이터)와 Exa/Tavily 웹 검색을 통해 up-to-date 분석을 제공한다.

**스크래치패드 디버깅**: 모든 도구 호출이 JSONL 파일에 로깅되어 에이전트의 추론 과정을 투명하게 파악할 수 있다.

## 스크래치패드 시스템

Dexter의 독특한 기능은 **스크래치패드**다. 각 쿼리마다 새 JSONL 파일이 `.dexter/scratchpad/`에 생성된다:

```
.dexter/scratchpad/
├── 2026-01-30-111400_9a8f10723f79.jsonl
├── 2026-01-30-143022_a1b2c3d4e5f6.jsonl
└── ...
```

JSONL 파일에는 세 종류의 항목이 있다:
- `init`: 원래 쿼리
- `tool_result`: 도구 호출 정보 (이름, 인수, 원시 결과, LLM 요약)
- `thinking`: 에이전트의 추론 단계

예시:
```json
{
  "type": "tool_result",
  "timestamp": "2026-01-30T11:14:05.123Z",
  "toolName": "get_income_statements",
  "args": {"ticker": "AAPL", "period": "annual", "limit": 5},
  "result": {...},
  "llmSummary": "애플 5년간 연간 손익계산서. 매출 $2740억에서 $3940억으로 성장"
}
```

이를 통해 에이전트가 어떤 데이터를 수집했고 어떻게 해석했는지 완전히 감사할 수 있다.

## 멀티 모델 지원 및 평가 시스템

**멀티 모델 지원**: `env.example`을 보면 다양한 모델을 선택할 수 있다:

```env
OPENAI_API_KEY=your-openai-api-key          # GPT-5, GPT-4o
ANTHROPIC_API_KEY=your-anthropic-api-key     # Claude (선택)
GOOGLE_API_KEY=your-google-api-key           # Gemini (선택)
XAI_API_KEY=your-xai-api-key                 # Grok (선택)
OPENROUTER_API_KEY=your-openrouter-api-key   # 라우터 (선택)
OLLAMA_BASE_URL=http://127.0.0.1:11434       # 로컬 Ollama (선택)
```

**평가(Eval) 시스템**: 금융 질문 데이터셋으로 에이전트 성능을 측정한다. LangSmith 추적과 LLM-as-judge 방식으로 정확도를 스코어링한다:

```bash
# 전체 데이터셋 평가
bun run src/evals/run.ts

# 랜덤 샘플 10개 평가
bun run src/evals/run.ts --sample 10
```

평가 실행 중 실시간 UI가 진행 상황, 현재 질문, 누적 정확도 통계를 표시한다.

## WhatsApp 게이트웨이

가장 흥미로운 기능 중 하나는 **WhatsApp 통합**이다. 자신에게 WhatsApp 메시지로 금융 질문을 보내면 Dexter가 분석하고 같은 채팅으로 답변한다:

```bash
# WhatsApp 계정 연결 (QR 코드 스캔)
bun run gateway:login

# 게이트웨이 시작
bun run gateway
```

WhatsApp 앱에서 자신에게 메시지를 보내면 된다. 모바일에서 금융 분석을 쉽게 접근할 수 있는 실용적인 인터페이스다.

## 기술 스택 및 설치

**기술 스택**: TypeScript 99.4%, Bun 런타임(v1.0+), Financial Datasets API(금융 데이터), Exa/Tavily(웹 검색), LangSmith(평가 추적, 선택).

**설치 및 실행**:

```bash
# 1. 클론
git clone https://github.com/virattt/dexter.git
cd dexter

# 2. Bun 설치 (없는 경우)
# macOS/Linux
curl -fsSL https://bun.com/install | bash
# Windows
powershell -c "irm bun.sh/install.ps1|iex"

# 3. 의존성 설치
bun install

# 4. 환경 변수 설정
cp env.example .env
# .env 편집:
# OPENAI_API_KEY=요청 필수
# FINANCIAL_DATASETS_API_KEY=금융 데이터 (필수)
# EXASEARCH_API_KEY=웹 검색 (권장)

# 5. 실행
bun start         # 인터랙티브 모드
bun dev           # 개발 모드 (watch)
```

**Financial Datasets API KEY**: `financialdatasets.ai`에서 취득. 소득계산서, 재무상태표, 현금흐름표 등 기관급 재무 데이터에 접근하는 핵심 API다.

**Exa API KEY**: `exa.ai`에서 취득 (선택). 웹 검색을 통해 최신 뉴스와 분석 리포트도 포함할 수 있다. Exa 없이는 Tavily가 폴백으로 사용된다.

## 마치며

Dexter는 금융 리서치의 지루한 부분을 자동화한다. 기업 재무제표 수집, 여러 분기 비교, 경쟁사 분석 등 반복적인 데이터 수집 작업을 AI 에이전트에게 맡기고, 인간은 인사이트 도출과 투자 결정에 집중할 수 있다.

스크래치패드 시스템으로 에이전트의 모든 추론 과정을 감사할 수 있어 '블랙박스'가 아니라 투명한 분석 도구로 사용할 수 있다. TypeScript + Bun의 빠른 런타임과 21.2k stars의 커뮤니티 지지가 이 도구의 성장 가능성을 보여준다.

⚠️ 면책조항: 이 도구는 리서치 목적이며 투자 조언을 제공하지 않는다. 투자 결정은 항상 독립적인 판단과 전문가 자문을 기반으로 해야 한다.

## 소개

Impeccable은 pbakaus가 공개한 **AI 에이전트용 프론트엔드 디자인 명령어 시스템**입니다. Apache 2.0 라이선스로 공개되어 있으며, 공식 사이트는 [impeccable.style](https://impeccable.style)입니다. 1개의 스킬 파일과 18개의 slash command로 구성된 단순한 구조지만, AI 에이전트에게 **프로 프론트엔드 디자이너의 어휘와 판단 기준**을 가르치는 것이 핵심 목표입니다.

AI가 코드를 생성할 때 디자인적으로 어설픈 결과물을 만드는 이유 중 하나는 "좋은 디자인"의 구체적인 기준을 모르기 때문입니다. Impeccable은 이 문제를 구체적인 디자인 패턴, 안티패턴, 비판 기준의 언어로 해결합니다. `/impeccable craft`로 시작해 `/audit`, `/critique`, `/polish` 같은 명령어를 사용하면 AI가 프로 디자이너처럼 UI를 분석하고 개선합니다.

디자인 어휘(vocabulary)의 중요성을 강조합니다. "버튼이 크다"라는 말 대신 "버튼의 터치 타겟이 44px 미니멈을 초과해 접근성 가이드라인을 충족한다"처럼 정확한 어휘가 AI의 디자인 품질을 좌우합니다.

## 주요 기능

- **/impeccable craft**: 처음부터 디자인 어휘를 갖춘 UI 컴포넌트를 생성합니다. AI가 프로 디자이너의 기준으로 선택을 하며 코드를 작성합니다.
- **/audit**: 기존 UI를 체계적으로 감사합니다. 접근성, 반응형, 색상 대비, 타이포그래피, 간격 등 주요 디자인 기준으로 문제를 식별합니다.
- **/critique**: UI를 프로 디자이너의 시각으로 비판적으로 평가합니다. "무엇이 문제인가"와 "왜 그것이 문제인가"를 구체적으로 설명합니다.
- **/polish**: 작동하지만 디자인이 어설픈 UI를 다듬습니다. 미세한 간격, 음영, 타이포그래피 개선으로 "프로답게" 만듭니다.
- **/distill**: 복잡한 UI를 핵심 요소만 남겨 단순화합니다. "무엇을 제거해도 기능이 유지되는가"를 판단합니다.
- **안티패턴 라이브러리**: 18개 명령어에 걸쳐 "피해야 할 패턴"이 명시됩니다. AI가 흔한 디자인 실수를 사전에 방지합니다.

### 18개 명령어 체계 — 디자인 워크플로우 전체 커버

```
생성 명령어:
  /craft      - 새 디자인 생성 (어휘 기반)
  /variant    - 다양한 디자인 변형 생성
  /prototype  - 빠른 프로토타입 생성

분석 명령어:
  /audit      - 체계적 감사 (접근성, 반응형, 색상 등)
  /critique   - 전문가 비평
  /compare    - 두 디자인 비교 분석

개선 명령어:
  /polish     - 세밀한 개선
  /refactor   - 구조적 재설계
  /simplify   - 과복잡성 제거
  /distill    - 핵심만 남기기

특화 명령어:
  /a11y       - 접근성 전문 감사 (WCAG 기준)
  /motion     - 애니메이션/전환 효과 설계
  /responsive - 반응형 디자인 최적화
  /darkmode   - 다크모드 변환/최적화
  /tokens     - 디자인 토큰 추출/정리
  /handoff    - 개발자 핸드오프 준비
  /document   - 디자인 문서화
  /spec       - 디자인 스펙 작성
```

### /audit — 체계적 디자인 감사의 예

```
사용자: /audit
[현재 컴포넌트 코드 붙여넣기]

Impeccable 응답:
접근성:
  ⚠️ 버튼에 aria-label 없음 — 스크린리더가 "버튼"만 읽음
  ✅ 색상 대비 4.8:1 — WCAG AA 기준 충족
  ❌ 포커스 인디케이터 없음 — 키보드 사용자 탐색 불가

타이포그래피:
  ⚠️ 3가지 이상의 폰트 사이즈 혼용 — 계층 구조 모호
  ❌ 줄 간격 1.2 — 가독성을 위해 1.4-1.6 권장

간격:
  ✅ 8px 그리드 시스템 일관성 유지
  ⚠️ 모바일에서 터치 타겟 36px — 44px 미니멈 미달
```

## 기술 스택 / 아키텍처

| 항목 | 내용 |
|------|------|
| 구성 | 1 스킬 파일 + 18 slash commands |
| 플랫폼 | Claude Code, Claude.ai |
| 공식 사이트 | impeccable.style |
| 라이선스 | Apache 2.0 |

## 설치 / 사용법

```bash
# 저장소 클론
git clone https://github.com/pbakaus/impeccable
cd impeccable

# 스킬 파일을 프로젝트에 복사
cp -r skills/ /path/to/project/skills/
# 또는 commands/ 폴더를 .claude/ 하위에 복사

# Claude Code에서 사용
/impeccable craft "소셜 미디어 공유 버튼 그룹"

# 기존 코드 감사
# 코드를 열고 /audit 실행
/audit

# 개선 제안 적용
/polish

# 접근성 전문 감사
/a11y
```

## 활용 사례 / 사용 시나리오

1. **코드-퍼스트 개발자의 디자인 품질 향상**: 디자인 지식이 부족한 개발자가 Impeccable을 사용하면 AI가 프로 디자이너의 기준을 대신 적용합니다. `/craft`로 생성하고 `/audit`으로 검증하는 흐름으로 디자인 품질이 크게 향상됩니다.

2. **디자인 리뷰 자동화**: PR 리뷰 과정에 `/audit` 명령어를 통합합니다. 코드 리뷰처럼 디자인 리뷰도 자동화해 접근성, 반응형, 타이포그래피 기준을 체계적으로 검증합니다.

3. **디자인 시스템 문서화**: `/tokens`와 `/document`로 기존 컴포넌트의 디자인 토큰을 추출하고 문서를 자동으로 생성합니다. 흩어진 스타일을 일관된 디자인 시스템으로 정리합니다.

## 결론

Impeccable은 "AI가 디자인을 잘 못한다"는 문제의 근본 원인을 정확하게 진단하고 해결합니다. 디자인 어휘와 판단 기준을 18개 명령어로 체계화해 AI에게 제공함으로써 프로 수준의 디자인 품질을 더 쉽게 달성합니다. 특히 디자인 리뷰와 접근성 감사 자동화에서 즉각적인 가치를 발휘합니다.

## 소개

Hive는 aden-hive 팀이 개발한 **프로덕션 환경용 AI 에이전트 런타임 하니스**입니다. GitHub에서 10,100개 이상의 별과 5,600개의 포크를 기록하며 에이전트 인프라 커뮤니티에서 빠르게 성장하고 있습니다. 목표를 자연어로 설명하면 **queen(코딩 에이전트)이 에이전트 그래프와 연결 코드를 자동으로 생성**하고, 실행 중에는 하니스가 상태 격리, 체크포인트 복구, 비용 제어, 실시간 관찰성을 관리합니다.

"에이전트를 만들고 싶지만 에이전트 인프라는 복잡하다"는 문제를 해결하기 위해 설계되었습니다. 단순한 오케스트레이터가 아니라 에이전트가 실패했을 때 자동으로 그래프를 발전시키고 재배포하는 **자가 치유(self-healing)** 메커니즘을 갖추고 있습니다. 내장된 브라우저 제어, 자격증명 관리, 병렬 실행 기능은 에이전트를 실제 업무에 투입할 때 필요한 요소들입니다.

커뮤니티 사이트 HoneyComb(honeycomb.open-hive.com)에서는 사용자들이 자동화 중인 직업들을 공유하고 "AI가 이 직업을 대체할 가능성"에 대한 예측 마켓을 운영합니다(실제 금전 거래 없이 컴퓨팅 토큰 기반).

## 주요 기능

- **자동 에이전트 그래프 생성**: 목표를 자연어로 설명하면 queen(코딩 에이전트)이 에이전트 그래프와 연결 코드를 생성합니다. 직접 그래프를 코딩할 필요가 없습니다.
- **체크포인트 기반 크래시 복구**: 에이전트가 실패하면 실패 데이터를 캡처하고, 그래프를 자동으로 발전시켜 재배포합니다.
- **비용 제어**: 팀, 에이전트, 워크플로우 단위로 예산 한도를 설정하고 실시간 비용 추적과 알림을 제공합니다. 예산 초과 시 모델 다운그레이드 정책도 구성 가능합니다.
- **Human-in-the-Loop 노드**: 파이프라인 중간에 사람의 검토·승인이 필요한 단계를 명시적으로 삽입할 수 있습니다.
- **내장 브라우저 제어**: 별도 Playwright 설정 없이 에이전트가 웹 인터페이스와 상호작용할 수 있습니다.
- **병렬 실행**: 독립적인 에이전트 서브태스크를 병렬로 실행해 처리 속도를 높입니다.

### 자가 치유 아키텍처 — 실패를 학습으로 전환

일반적인 에이전트 프레임워크는 에이전트가 실패하면 오류를 반환하고 종료합니다. Hive는 다릅니다. 에이전트가 실패하면 **하니스가 실패 데이터를 자동으로 캡처**하고, queen(코딩 에이전트)에게 전달해 에이전트 그래프를 수정·개선한 뒤 다시 배포합니다. 이 흐름이 자동화되어 있어 운영자의 직접 개입 없이도 시스템이 안정화됩니다. 장기간 실행되는 비즈니스 자동화 파이프라인에서 특히 가치 있는 기능입니다.

### 비용 제어 — 에이전트 예산 관리

LLM API 비용은 예측하기 어렵습니다. Hive는 팀 단위, 에이전트 단위, 워크플로우 단위로 지출 한도, 스로틀, 자동 모델 다운그레이드 정책을 설정할 수 있습니다. 실시간 비용 추적과 알림이 내장되어 있어 예산 초과를 사전에 방지합니다. 기업 환경에서 AI 비용 거버넌스를 구현하기에 적합한 구조입니다.

## 기술 스택 / 아키텍처

| 항목 | 내용 |
|------|------|
| 언어 | Python (JS/TS SDK 로드맵) |
| 배포 | 로컬, Docker |
| 에이전트 기능 | 브라우저 제어, 크레덴셜 관리, 병렬 실행 |
| 관찰성 | 실시간 모니터링, 비용 추적 |
| 라이선스 | 명시된 라이선스 |
| 별 | 10,100+ |

아키텍처 흐름:  
`목표 입력 → queen(코딩 에이전트)이 에이전트 그래프 생성 → 하니스 실행 감독 → 실패 시 그래프 진화 → 재배포`

## 설치 / 사용법

```bash
# macOS/Linux 빠른 시작
git clone https://github.com/aden-hive/hive
cd hive
bash scripts/quickstart.sh
```

```powershell
# Windows PowerShell
git clone https://github.com/aden-hive/hive
cd hive
.\scripts\quickstart.ps1
```

전체 문서는 [docs.adenhq.com](https://docs.adenhq.com/)에서 확인할 수 있습니다. `docs/developer-guide.md`에 상세한 API 레퍼런스와 시작 튜토리얼이 있습니다.

## 활용 사례 / 사용 시나리오

1. **비즈니스 프로세스 자동화**: 웹 기반 내부 시스템(ERP, CRM, 이메일)과 상호작용하는 장기 실행 에이전트를 구축합니다. 체크포인트 복구 덕분에 네트워크 오류나 API 실패에도 자동으로 이어서 실행됩니다.

2. **AI 에이전트 팀 운영**: 여러 전문 에이전트(데이터 수집, 분석, 보고서 작성)를 병렬로 실행하고 비용을 팀 단위로 추적·제어합니다. 기업 환경에서 AI 거버넌스와 예산 관리가 필요한 경우에 특히 유용합니다.

3. **자동화 워크플로우 프로토타이핑**: 자연어로 목표를 설명하면 queen이 에이전트 그래프를 자동 생성하므로, 에이전트 인프라 전문 지식 없이도 빠르게 자동화 PoC를 구현할 수 있습니다.

## 결론

Hive는 에이전트를 "실험"에서 "운영"으로 끌어올리기 위해 필요한 인프라 요소들을 패키지로 제공합니다. 특히 자동 복구, 비용 제어, 관찰성은 프로덕션 배포에서 필수적인 요소들입니다. 현재 Python SDK만 안정적으로 지원되며 JS/TS는 로드맵 단계이므로 TypeScript 기반 프로젝트는 조금 더 기다려야 합니다. 에이전트 파이프라인을 지속적으로 실행해야 하는 팀에게 강하게 추천합니다.

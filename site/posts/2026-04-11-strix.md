## 소개

Strix는 실제 해커처럼 동작하는 자율 AI 보안 에이전트 프레임워크입니다. 코드를 동적으로 실행하고, 취약점을 발견하며, 실제 개념 증명(PoC)을 통해 취약점을 검증합니다. 정적 분석 도구의 가짜 양성(false positive)이나 수동 침투 테스트의 오랜 대기 없이, 개발자와 보안 팀이 빠르고 정확한 보안 테스트를 수행할 수 있도록 설계되었습니다.

GitHub에서 23.4k 스타를 획득한 Strix는 Python 91.6%, Jinja 4.2%, Shell 2.7%로 구성되어 있으며 Apache-2.0 라이선스로 배포됩니다. 27명의 기여자가 활발하게 개발하고 있으며, GitHub Actions와의 원활한 CI/CD 통합이 최근 새로운 기능으로 추가되었습니다.

## 주요 기능

Strix는 보안 테스트의 전 과정을 자동화하는 포괄적인 기능을 제공합니다.

**에이전틱 보안 도구:** 완전한 HTTP 프록시(요청/응답 조작 및 분석), 브라우저 자동화(XSS/CSRF/인증 흐름 테스트용 멀티탭), 대화형 셸(터미널 환경), Python 런타임(커스텀 익스플로잇 개발), 정찰(자동 OSINT 및 공격 표면 매핑), 코드 분석(정적/동적 분석)이 기본 포함됩니다.

**종합적인 취약점 탐지:** 접근 제어(IDOR, 권한 상승, 인증 우회), 인젝션 공격(SQL/NoSQL/커맨드 인젝션), 서버 사이드(SSRF/XXE/역직렬화), 클라이언트 사이드(XSS/프로토타입 오염/DOM 취약점), 비즈니스 로직(레이스 컨디션/워크플로우 조작), 인증(JWT 취약점/세션 관리), 인프라(잘못된 설정/노출된 서비스) 등을 망라합니다.

**에이전트 그래프:** 다양한 공격 유형과 자산에 특화된 분산 워크플로우, 빠른 종합 커버리지를 위한 병렬 실행, 에이전트 간 발견 사항 공유와 동적 조정이 이루어집니다.

## CI/CD 통합 심층 분석

Strix의 가장 혁신적인 기능 중 하나는 GitHub Actions와의 원활한 통합입니다. PR이 올라올 때마다 자동으로 보안 테스트를 실행하고 취약한 코드가 프로덕션에 도달하기 전에 차단할 수 있습니다.

```yaml
name: strix-penetration-test

on:
  pull_request:

jobs:
  security-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
        with:
          fetch-depth: 0

      - name: Install Strix
        run: curl -sSL https://strix.ai/install | bash

      - name: Run Strix
        env:
          STRIX_LLM: ${{ secrets.STRIX_LLM }}
          LLM_API_KEY: ${{ secrets.LLM_API_KEY }}
        run: strix -n -t ./ --scan-mode quick
```

CI의 PR 실행에서 Strix는 자동으로 변경된 파일에 스코프를 맞춥니다. `--non-interactive (-n)` 플래그로 헤드리스 모드 실행이 가능하며, 취약점 발견 시 비정상 종료 코드로 파이프라인을 중단시킵니다.

## 스텔스 스캔 및 고급 시나리오

Strix는 다양한 스캔 시나리오를 지원합니다. 블랙박스 웹 애플리케이션 평가, 로컬 코드베이스 화이트박스 스캔, GitHub 저장소 보안 리뷰, 그리고 인증된 그레이박스 테스트까지 가능합니다.

```bash
# 블랙박스 웹 애플리케이션 스캔
strix --target https://your-app.com

# 인증된 그레이박스 테스트
strix --target https://your-app.com \
  --instruction "Perform authenticated testing using credentials: user:pass"

# 멀티 타깃 테스트 (소스 코드 + 배포된 앱)
strix -t https://github.com/org/app -t https://your-app.com

# 소스 인식 화이트박스 스캔
strix --target ./app-directory --scan-mode standard

# 비즈니스 로직 취약점에 집중
strix --target api.your-app.com \
  --instruction "Focus on business logic flaws and IDOR vulnerabilities"
```

스캔 결과는 `strix_runs/<run-name>` 디렉토리에 자동 저장되며, PoC와 재현 단계가 포함된 검증된 취약점 보고서가 생성됩니다.

## 기술 스택 및 아키텍처

Strix는 Python 기반의 멀티 에이전트 오케스트레이션 아키텍처를 채택합니다. Docker 컨테이너 기반의 샌드박스 환경에서 에이전트가 실행되어 안전한 격리를 보장합니다.

**지원 LLM 모델:**
- OpenAI GPT-5.4 (`openai/gpt-5.4`)
- Anthropic Claude Sonnet 4.6 (`anthropic/claude-sonnet-4-6`)
- Google Gemini 3 Pro Preview (`vertex_ai/gemini-3-pro-preview`)

LiteLLM을 활용하여 Vertex AI, Bedrock, Azure, 로컬 모델(Ollama, LMStudio) 등 다양한 LLM 프로바이더를 지원합니다. 구성은 `~/.strix/cli-config.json`에 자동 저장되어 반복 입력이 필요 없습니다.

오픈소스 프로젝트 LiteLLM, Caido, Nuclei, Playwright, Textual의 견고한 기반 위에 구축되었습니다.

## 설치 및 사용법

**필수 조건:** Docker(실행 중), 지원되는 LLM 프로바이더 API 키

```bash
# Strix 설치
curl -sSL https://strix.ai/install | bash

# AI 프로바이더 설정
export STRIX_LLM="openai/gpt-5.4"
export LLM_API_KEY="your-api-key"

# 선택적 설정
export LLM_API_BASE="your-api-base-url"  # 로컬 모델용
export PERPLEXITY_API_KEY="your-api-key"  # 검색 기능
export STRIX_REASONING_EFFORT="high"  # 추론 강도 (high/medium/low)

# 첫 번째 보안 평가 실행
strix --target ./your-app-directory

# GitHub 저장소 보안 리뷰
strix --target https://github.com/org/repo

# 웹 애플리케이션 블랙박스 평가
strix --target https://your-app.com
```

첫 실행 시 샌드박스 Docker 이미지를 자동으로 풀합니다. [Strix 클라우드 플랫폼](https://app.strix.ai)에서는 무료 계정 생성 후 저장소와 도메인을 연결하여 몇 분 안에 침투 테스트를 시작할 수 있습니다.

## 마치며

Strix는 보안 테스트의 패러다임을 바꾸는 도구입니다. 주 단위로 시간이 걸리던 침투 테스트를 몇 시간으로 단축하고, PR마다 자동으로 보안 취약점을 스캔하는 CI/CD 파이프라인을 구축할 수 있습니다.

특히 "Bug Bounty 자동화", "애플리케이션 보안 테스트", "신속한 침투 테스트" 등 다양한 사용 사례를 지원하며, 가짜 양성 없이 실제 PoC로 검증된 취약점만 보고합니다. 단, 본인이 소유하거나 테스트 권한을 받은 앱에만 사용해야 합니다. 오픈소스이지만 엔터프라이즈급 컨트롤(SSO/SAML/OIDC, 컴플라이언스 보고서, 전용 지원)도 제공됩니다.

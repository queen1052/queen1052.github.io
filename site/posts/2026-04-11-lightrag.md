## 소개

검색 증강 생성(RAG)은 LLM이 외부 지식을 참조해 더 정확한 답변을 생성하는 핵심 기술입니다. 그러나 기존 RAG 시스템은 텍스트 유사도 기반 검색에 머물러 문서 간 관계와 전체적인 지식 구조를 놓치는 한계가 있었습니다.

**LightRAG**는 홍콩대학교(HKUDS) 연구팀이 개발한 그래프 기반 접근법으로 이 문제를 해결합니다. 지식 그래프를 활용한 엔티티-관계 추출과 이중 레벨 검색 전략으로 NaiveRAG를 일관되게 능가하며, EMNLP 2025에 채택된 논문([arXiv:2410.05779](https://arxiv.org/abs/2410.05779))의 구현체입니다. Stars 32,900개, Fork 4,700개, 기여자 239명의 활발한 오픈 소스 프로젝트입니다.

## 핵심 기능 1: 그래프 기반 이중 레벨 검색

LightRAG의 차별화된 강점은 텍스트 유사도 검색을 넘어선 **지식 그래프 기반 검색**입니다.

**인덱싱 단계**:
1. LLM이 문서에서 엔티티(사람, 장소, 개념 등)와 관계를 추출
2. 추출된 정보로 지식 그래프 구성
3. 엔티티와 관계를 벡터 임베딩으로 변환해 저장

**쿼리 단계**:
- **Local 모드**: 특정 엔티티 중심의 세부 정보 쿼리
- **Global 모드**: 전체 문서에 걸친 주제·관계 쿼리
- **Hybrid 모드** (권장): Local + Global 결합
- **Mix 모드**: 리랭커 활성화 시 기본값으로 최고 성능

성능 비교 (농업·CS·법률·혼합 도메인 평균):
- NaiveRAG 대비 종합성: **67.6% 우위**
- NaiveRAG 대비 다양성: **76.4% 우위**
- GraphRAG 대비 다양성: **77.2% 우위**

## 핵심 기능 2: LightRAG 서버와 WebUI

**LightRAG Server**는 Web UI와 REST API를 제공하는 완전한 서비스 레이어입니다.

주요 서버 기능:
- 문서 인덱싱 웹 인터페이스
- 지식 그래프 시각화 (중력 레이아웃, 노드 쿼리, 서브그래프 필터링)
- 간단한 RAG 쿼리 인터페이스
- **Ollama 호환 인터페이스**: Open WebUI 등 Ollama 기반 AI 챗봇에서 LightRAG를 Ollama 모델처럼 사용 가능

LLM 요구 사항:
- 파라미터 수: 32B+ 권장
- 컨텍스트 길이: 32KB 이상 (64KB 권장)
- 인덱싱 중 추론 모델 비권장
- 쿼리 단계에서는 인덱싱보다 강력한 모델 사용 권장

## 기술 스택 및 아키텍처

```
LightRAG/
├── lightrag/          # 핵심 라이브러리
├── lightrag_webui/    # Bun + TypeScript 프론트엔드
├── docs/              # 설치/설정 가이드
├── examples/          # OpenAI, Ollama 등 예제
├── scripts/           # 설정 마법사
├── Dockerfile
├── docker-compose.yml
└── Makefile           # make dev, make env-base 등

```

- **언어**: Python 80.8%, TypeScript 13.4%, Shell 5.3%
- **라이선스**: MIT
- **패키지**: `lightrag-hku` (PyPI)
- **스토리지**: PostgreSQL, MongoDB, Neo4j, OpenSearch, SQLite

임베딩 모델 선택 시 주의: 문서 인덱싱 전에 확정해야 하며, 변경 시 벡터 테이블 재생성 필요. `BAAI/bge-m3` 또는 `text-embedding-3-large` 권장.

## 설치 및 사용법

### uv로 서버 설치 (권장)

```bash
uv tool install "lightrag-hku[api]"
```

### Docker Compose로 실행

```bash
git clone https://github.com/HKUDS/LightRAG.git
cd LightRAG
cp env.example .env  # LLM·임베딩 설정 편집
docker compose up
```

### 설정 마법사

```bash
make env-base      # LLM, 임베딩, 리랭커 필수 설정
make env-storage   # 스토리지 백엔드 선택 (선택적)
make env-server    # 서버 포트, 인증, SSL (선택적)
make env-security-check  # 보안 감사
```

### 빠른 시작 (코어 API)

```python
import asyncio
from lightrag import LightRAG, QueryParam
from lightrag.llm.openai import gpt_4o_mini_complete, openai_embedding

rag = LightRAG(
    working_dir="./dickens",
    llm_model_func=gpt_4o_mini_complete,
    embedding_func=openai_embedding,
)

# 문서 삽입
with open("./book.txt") as f:
    await rag.ainsert(f.read())

# 쿼리 (혼합 모드)
result = await rag.aquery(
    "주인공의 변화를 분석해줘",
    param=QueryParam(mode="hybrid")
)
```

## 마치며

LightRAG는 학술 연구와 실용적 구현을 성공적으로 결합한 RAG 시스템입니다. EMNLP 2025 채택으로 방법론의 학술적 검증을 받았고, 32,900개의 Star와 239명의 기여자가 보여주듯 커뮤니티의 신뢰도 높습니다.

지식 그래프 기반 접근법은 단순 벡터 검색이 놓치는 문서 간 관계와 구조적 지식을 포착해, 복잡한 멀티홉 질문이나 종합적인 분석 작업에서 기존 RAG를 크게 능가합니다. RAG 시스템을 구축하거나 개선하려는 개발자라면 반드시 검토할 후보입니다.

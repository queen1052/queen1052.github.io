## 소개

TREK은 mauriceboe가 개발한 **자체 호스팅 여행 계획 웹 앱**입니다. 원래 NOMAD라는 이름으로 시작했다가 TREK으로 재명명되었습니다. GitHub에서 3,900개의 별을 받은 인기 프로젝트로, 팀이나 가족과 함께 여행 일정을 계획하고 실시간으로 공유하는 도구입니다.

Node.js 22 + Express 백엔드, React 18 + Vite + Tailwind 프론트엔드, SQLite 데이터베이스의 깔끔한 풀스택 구성입니다. WebSocket으로 실시간 공동 편집을 지원하고, Leaflet 지도로 여행 경로를 인터랙티브하게 시각화합니다. PWA(Progressive Web App)로 모바일 설치를 지원하며, 기업 환경을 위한 SSO(OIDC) + MFA 인증도 통합되어 있습니다.

JWT 인증, Zustand 상태 관리, SQLite의 경량 데이터베이스 조합으로 작은 서버(Raspberry Pi 포함)에서도 효율적으로 실행됩니다. AGPL-3.0 라이선스로 공개되어 소규모 그룹이 프라이버시 걱정 없이 자체 서버에서 운영할 수 있습니다.

## 주요 기능

- **실시간 협업 편집**: WebSocket으로 여러 사용자가 동시에 여행 계획을 편집합니다. 변경사항이 실시간으로 반영되어 채팅보다 자연스러운 공동 계획 수립이 가능합니다.
- **인터랙티브 지도**: Leaflet 기반 지도에서 방문할 장소를 핀으로 표시하고, 경로를 선으로 연결합니다. 지도를 직접 탐색하면서 여행 경로를 설계합니다.
- **일정 관리**: 날짜별 일정 타임라인, 숙소 예약 정보, 이동 수단을 체계적으로 관리합니다. 체크리스트 형태로 준비 목록을 추적합니다.
- **예산 추적**: 여행 전체 예산을 설정하고 각 항목(숙소, 교통, 식사, 활동)별 지출을 기록합니다. 실시간 예산 진행 상황을 대시보드로 확인합니다.
- **짐 목록(Packing List)**: 공동 짐 목록을 만들고 팀원들이 각자 담당 항목을 체크합니다. "누가 텐트를 가져오는가"를 명확하게 관리합니다.
- **PWA + 모바일**: Progressive Web App으로 홈 화면에 설치할 수 있습니다. 오프라인에서도 기본 기능을 사용하고 온라인 복구 시 자동 동기화됩니다.
- **SSO (OIDC) + MFA**: 기업이나 조직 환경에서 기존 ID 제공자(Google Workspace, Okta 등)로 인증합니다. MFA(다중 인증)으로 보안을 강화합니다.

### 실시간 협업 — WebSocket 아키텍처

```
사용자 A (서울)              사용자 B (부산)
     |                            |
편집: "제주 한라산 추가"           편집: "우도 스노클링 추가"
     |                            |
     ↓                            ↓
   WebSocket                  WebSocket
         ↘                 ↙
        TREK 서버 (Express)
              |
        SQLite 저장 + 브로드캐스트
              |
        실시간 동기화 ← → 양쪽 화면 업데이트
```

편집 충돌은 서버 기반 최종 쓰기(last-write-wins) 또는 OT(Operational Transformation) 방식으로 해결됩니다.

## 설치 / 사용법

```bash
# 저장소 클론
git clone https://github.com/mauriceboe/TREK
cd TREK

# 의존성 설치
npm install

# 환경 변수 설정
cp .env.example .env
# JWT_SECRET, DATABASE_PATH, PORT 등 설정
# SSO 사용 시: OIDC_ISSUER, OIDC_CLIENT_ID, OIDC_CLIENT_SECRET

# 개발 서버
npm run dev

# 프로덕션 빌드
npm run build
npm start

# Docker로 배포
docker compose up -d
```

Docker Compose 설정:
```yaml
services:
  trek:
    image: mauriceboe/trek:latest
    ports:
      - "3000:3000"
    volumes:
      - ./data:/app/data  # SQLite 데이터 영구 저장
    environment:
      - JWT_SECRET=your-secret-key
      - PORT=3000
```

## 활용 사례 / 사용 시나리오

1. **가족 여행 계획**: 가족 구성원 모두가 TREK에 접속해 실시간으로 여행 일정을 수정합니다. 아이들이 가고 싶은 장소를 지도에 직접 핀으로 표시하고 토론합니다. 예산 추적으로 가족 여행 전체 지출을 투명하게 관리합니다.

2. **기업 출장 관리**: SSO(OIDC) 통합으로 회사 계정으로 로그인하고 MFA로 보안을 유지합니다. 팀 출장 일정, 숙소, 항공권 정보를 한 곳에서 관리하고 경비 정산에 활용합니다.

3. **여행 커뮤니티 운영**: 작은 커뮤니티 여행 클럽이 자체 서버에 TREK을 배포해 회원들의 여행을 공동으로 계획합니다. AGPL-3.0으로 완전 자체 호스팅이 가능하며 회원 데이터가 외부로 나가지 않습니다.

## 결론

TREK은 현대적인 기술 스택(Node.js 22, React 18, WebSocket, PWA)으로 여행 계획의 핵심 문제(공동 편집, 지도 시각화, 예산 관리)를 깔끔하게 해결합니다. 3.9k 별과 지속적인 유지보수가 프로젝트의 건강성을 보여줍니다. 상업 여행 앱의 광고와 데이터 수집 없이 완전한 프라이버시로 그룹 여행을 계획하고 싶은 사람에게 최선의 선택입니다.

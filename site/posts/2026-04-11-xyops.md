## xyOps란?

**xyOps™**는 잡 스케줄링, 워크플로우 자동화, 서버 모니터링, 스마트 알림, 인시던트 대응을 하나의 통합 플랫폼에서 제공하는 차세대 운영 자동화 시스템입니다. GitHub에서 3,900개 이상의 스타를 받았으며 JavaScript 92.8%로 구현되어 BSD-3-Clause 라이선스로 공개되었습니다.

"데이터, 자유, 가시성을 포기하지 않고 자동화 스택을 제어하고 싶은 개발자와 운영팀을 위해 만들어졌다"는 철학 아래, 숨겨진 기능 없이 텔레메트리도 없는 순수 오픈소스 플랫폼입니다. BDS-3 라이선스로 영구적으로 오픈소스를 유지하겠다는 Longevity Pledge도 공식 문서화되어 있습니다.

## 주요 기능

xyOps의 핵심 기능들을 살펴보겠습니다.

**잡 스케줄링 재발명**: 단순 크론을 훨씬 뛰어넘는 잡 스케줄링을 제공합니다. 복잡한 의존성, 재시도 정책, 실행 이력, 실시간 로그 스트리밍을 지원합니다.

**시각적 워크플로우 빌더**: 그래픽 워크플로우 편집기로 이벤트, 트리거, 액션, 모니터를 드래그앤드롭으로 연결하여 의미 있는 파이프라인을 구성합니다.

**플릿 모니터링**: 5대든 5,000대든 모든 서버를 하나의 뷰로 모니터링합니다. 실시간 스냅샷(모든 프로세스, CPU 부하, 네트워크 연결)을 캡처합니다.

**스마트 알림**: 완전한 커스터마이징과 복잡한 트리거를 지원하는 윤택한 알림 시스템. 알림 이메일에 해당 서버에서 실행 중인 잡 목록이 포함됩니다.

**통합 피드백 루프**: xyOps의 핵심 차별점은 모든 요소가 연결된다는 것입니다. 알림 → 서버 스냅샷 → 잡 실패 → 컨텍스트가 포함된 티켓 자동 생성까지 하나의 시스템에서 추적할 수 있습니다.

## 통합 인시던트 대응

xyOps가 단순 스케줄러와 다른 가장 큰 차이점은 **통합된 인시던트 대응 흐름**입니다.

전통적인 스택에서는 모니터링 도구(Datadog/Grafana), 잡 스케줄러(Airflow/Jenkins), 티켓 시스템(Jira/ServiceNow)이 각각 분리되어 있어 "이 스파이크가 어떤 잡 때문에 생긴 거야?" 같은 질문에 답하려면 여러 툴을 전전해야 했습니다.

xyOps에서는:
1. 알림 이메일에 해당 서버에서 실행 중인 잡 목록 포함
2. 원클릭으로 모든 프로세스·CPU·네트워크를 보여주는 스냅샷 확인
3. 잡 실패 시 완전한 컨텍스트(로그, 이력, 연동 메트릭)를 포함한 티켓 자동 오픈

이 통합 피드백 루프 덕분에 감지부터 해결까지 시스템을 벗어나지 않아도 됩니다.

## 기술 스택 및 아키텍처

**언어**: JavaScript 92.8%, CSS 5.8%

**핵심 아키텍처**:
- Node.js 기반 백엔드
- 마스터-위성(Satellite) 구조로 분산 플릿 관리
- HTTP Request 플러그인을 통한 외부 시스템 연동
- Docker 컨테이너 지원

**플러그인 시스템**: HTTP Request, cron 표현식 기반 스케줄러, 알림 채널 등이 플러그인 방식으로 확장 가능합니다.

**보안**: CSRF 토큰 시스템, 위협 모델 문서화(THREAT_MODEL.md), 보안 오버뷰(SECURITY_OVERVIEW.md) 제공.

## 설치 및 사용법

**Docker 실행** (가장 빠른 방법):
```bash
docker pull pixlcore/xyops:latest
docker run -d \
  --name xyops \
  -p 3012:3012 \
  -v /opt/xyops/data:/opt/xyops/data \
  pixlcore/xyops:latest
```

**소스에서 빌드** (개발):
```bash
git clone https://github.com/pixlcore/xyops.git
cd xyops
npm install
node bin/build.js dev
echo '{ "secret_key": "test" }' > conf/overrides.json
bin/debug.sh
```

설치 후 `http://localhost:3012`에서 웹 UI에 접속합니다.

**잡 설정 (JSON 예시)**:
```json
{
  "id": "daily_backup",
  "title": "Daily Database Backup",
  "enabled": true,
  "schedule": "0 2 * * *",
  "target": {
    "server": "db-server-01",
    "command": "/opt/scripts/backup.sh"
  },
  "notifications": {
    "on_fail": ["ops-team@company.com"]
  }
}
```

전체 문서는 `https://docs.xyops.io/`에서 확인할 수 있으며, 앱 내 사이드바의 "Documentation"에서도 접근 가능합니다.

## 가격 정책

- **무료 티어**: 개인 셀프호스팅. 모든 앱 기능, 커뮤니티 지원, 영구 오픈소스
- **Professional 플랜**: 프로덕션 설치용. 전문 지원, 비공개 티켓 시스템, 24시간 응답
- **Enterprise 플랜**: 대기업용. SSO, 에어갭(air-gapped) 설치 지원, 1시간 응답, 라이브 채팅

## 마치며

xyOps는 분산된 DevOps 툴 스택에 지친 팀을 위한 강력한 대안입니다. "Alert fires → 실행 중 잡 확인 → 서버 스냅샷 → 티켓 생성"으로 이어지는 인시던트 대응 흐름이 하나의 플랫폼에서 완결되는 경험은 다른 도구에서 느끼기 힘든 매력입니다. BSD-3 라이선스와 Longevity Pledge는 긴 시간 동안 인프라 핵심 컴포넌트로 활용하기에 충분한 신뢰를 제공합니다.

## 소개

RAGflow는 infiniflow가 개발한 **기업급 오픈소스 RAG(Retrieval-Augmented Generation) 플랫폼**입니다. 단순 벡터 검색 기반 RAG를 넘어 **딥 문서 이해(Deep Document Understanding)**를 핵심 기술로 채택합니다. PDF, Word, Excel, PowerPoint, HTML, 이미지 등 다양한 문서 형식에서 레이아웃 구조, 표, 수식, 이미지를 정확하게 파이상해 검색 품질을 높입니다.

RAGflow의 차별점은 문서 파싱 품질에 있습니다. 기존 RAG 시스템은 PDF를 단순 텍스트로 변환해 청킹하는데, 이 과정에서 표의 데이터, 다단 레이아웃, 수식이 뒤섞입니다. RAGflow는 각 문서 요소(표, 그림 캡션, 제목 계층, 수식)를 구조적으로 인식하고 의미론적으로 청킹합니다.

`cloud.ragflow.io`에서 즉시 체험할 수 있으며, Docker 컴포즈 한 줄로 자체 호스팅도 가능합니다. React 프론트엔드와 Python 백엔드로 구현되어 있으며, 활발한 커뮤니티와 지속적인 커밋이 안정적인 유지보수를 보장합니다.

## 주요 기능

- **딥 문서 이해**: 단순 텍스트 추출을 넘어 PDF, Word, Excel, PPT의 레이아웃 구조, 표 데이터, 수식을 정확하게 인식합니다. 문서의 구조적 의미를 이해한 청킅으로 검색 정확도가 향상됩니다.
- **다중 문서 형식 지원**: PDF, Word (.docx), Excel (.xlsx), PowerPoint (.pptx), HTML, Markdown, 이미지(OCR 포함)를 지원합니다. 기업의 다양한 지식 소스를 통합합니다.
- **신뢰 가능한 인용 추적**: 답변에 사용된 정보의 출처(문서명, 페이지, 청그크)를 명시합니다. AI 답변을 원문과 대조 검증할 수 있어 기업 환경의 신뢰성 요구를 충족합니다.
- **다중 LLM 지원**: OpenAI GPT, Anthropic Claude, Google Gemini, Azure OpenAI, 로컬 Ollama 모델을 지원합니다. 클라우드와 온프레미스 혼합 배포가 가능합니다.
- **Agentic RAG**: 단순 Q&A를 넘어 멀티스텝 추론, 지식 그래프 탐색, 반복적 검색 개선(Iterative RAG)을 지원합니다.
- **Docker 자체 호스팅**: `docker compose up` 한 줄로 전체 스택을 배포합니다. 기업 데이터가 외부로 나가지 않는 완전한 온프레미스 배포를 지원합니다.

### 딥 문서 이해 — RAG 품질의 핵심

기존 RAG의 파싱 실패 사례:

```
[원본 표]
| 제품명 | Q1 매출 | Q2 매출 | 성장률 |
|--------|---------|---------|--------|
| 제품A  | 100만원 | 150만원 | 50%   |
| 제품B  | 200만원 | 180만원 | -10%  |

[기존 RAG 추출]
"제품명 Q1 매출 Q2 매출 성장률 제품A 100만원 150만원 50% 제품B 200만원 180만원 -10%"
→ 표 구조 손실, "제품B의 Q2 매출은?"에 오답

[RAGflow 추출]
{
  "type": "table",
  "headers": ["제품명", "Q1 매출", "Q2 매출", "성장률"],
  "rows": [
    ["제품A", "100만원", "150만원", "50%"],
    ["제품B", "200만원", "180만원", "-10%"]
  ]
}
→ 표 구조 보존, 정확한 Q&A 가능
```

### 인용 추적 — 신뢰 가능한 AI 답변

```json
{
  "answer": "2025년 Q2 총 매출은 330만원입니다.",
  "citations": [
    {
      "document": "Q2_사업보고서.pdf",
      "page": 5,
      "chunk": "표 3: 제품별 분기 매출",
      "text": "제품A Q2: 150만원, 제품B Q2: 180만원"
    }
  ]
}
```

## 기술 스택 / 아키텍처

| 항목 | 내용 |
|------|------|
| Frontend | React |
| Backend | Python |
| 배포 | Docker Compose |
| 벡터 DB | 내장 (또는 외부 Chroma/Milvus) |
| LLM 지원 | OpenAI, Claude, Gemini, Azure, Ollama |
| 클라우드 데모 | cloud.ragflow.io |
| 라이선스 | Apache-2.0 |

아키텍처:  
`문서 업로드 → 딥 문서 파싱 → 의미론적 청킹 → 벡터 저장 → 의미론적 검색 → LLM 답변 생성 → 인용 추적 반환`

## 설치 / 사용법

```bash
# Docker 자체 호스팅
git clone https://github.com/infiniflow/ragflow
cd ragflow

# 환경 변수 설정
cp .env.example .env
# OPENAI_API_KEY 또는 다른 LLM API 키 설정
# (선택) MILVUS_HOST, MILVUS_PORT for 외부 벡터 DB

# 실행
docker compose up -d

# 브라우저에서 http://localhost:80 접속
# 또는 기본 포트: http://localhost:9380

# 업그레이드
docker compose down
git pull
docker compose up -d
```

**클라우드 데모**: [cloud.ragflow.io](https://cloud.ragflow.io)에서 즉시 체험 가능 (무료 플랜 제공).

## 활용 사례 / 사용 시나리오

1. **기업 지식베이스 Q&A**: 사내 문서(정책, 매뉴얼, 보고서)를 RAGflow에 업로드하고 직원들이 자연어로 질문합니다. 인용 추적 덕분에 AI 답변의 근거를 원문에서 바로 확인합니다.

2. **법률/규정 문서 검색**: 계약서, 법령, 규정 문서를 관리하고 특정 조항을 자연어로 검색합니다. 딥 문서 이해가 법률 문서의 복잡한 구조(표, 리스트, 중첩 조항)를 정확하게 처리합니다.

3. **연구 논문 지식베이스**: 수백 편의 논문을 RAGflow에 인덱싱하고 "이 주제에서 가장 많이 인용된 방법론은?"처럼 자연어로 묻습니다. 출처 인용으로 검증 가능한 연구 지원을 받습니다.

## 결론

RAGflow는 기업 환경에서 RAG를 진지하게 배포하려는 팀의 좋은 출발점입니다. 딥 문서 이해, 인용 추적, 다양한 LLM 지원, Docker 자체 호스팅이 균형 잡힌 완성도를 보여줍니다. 클라우드 데모(cloud.ragflow.io)로 즉시 체험하고, 데이터 보안이 중요하면 Docker로 자체 호스팅하면 됩니다. 활발한 커밋 활동이 지속적인 기능 개선을 약속합니다.

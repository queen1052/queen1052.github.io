## 소개: 주머니 속 AI 에이전트

AI 코딩 에이전트의 가장 큰 제약 중 하나는 **노트북에 묶여 있다**는 것이다. 자리를 뜨면 작업이 멈춘다. 스마트폰에서 확인하려면 새로운 채팅을 시작해야 하고, 이전 맥락은 사라진다.

대부분의 "IM + 에이전트" 솔루션들은 이 문제를 새로운 에이전트를 만들거나(공식 CLI보다 성능 저하), 원격 샌드박스에서 실행하거나(내 환경이 아님), 짧은 대화만 지원하는(긴 작업에 불가) 방식으로 접근한다.

**pikiclaw**는 다른 선택을 한다: **공식 에이전트 CLI를 그대로 쓰되, 내 컴퓨터에서, 기존 메신저로 제어한다.**

```
당신 (Telegram / 飞书 / WeChat)
        |
        v
     pikiclaw
        |
        v
Claude Code / Codex / Gemini CLI
        |
        v
   내 컴퓨터
```

GitHub 스타 147개, TypeScript 95.5%, MIT 라이선스. `npx pikiclaw@latest` 한 줄로 시작한다.

## 주요 기능

**3개 IM 채널 동시 지원**: Telegram, 飞书(Feishu), 微信(WeChat). 하나 또는 전부 동시에 실행 가능.

**3개 에이전트 CLI 통합**: Claude Code, Codex CLI, Gemini CLI를 통합 드라이버 레지스트리를 통해 지원.

**웹 대시보드**: `http://localhost:3939`에 에이전트 콘솔이 열린다. 전체 대화 기록, 도구 사용 활동, thinking 트레이스, 플랜 진행 상황, 스트리밍 출력, 이미지 첨부, 입력 작성기까지 — Claude Code의 핵심 인터랙티브 경험을 브라우저에서 구현했다.

**스트리밍 미리보기**: 에이전트가 작업하는 동안 메신저에 실시간으로 스트리밍 업데이트가 온다.

**태스크 큐 + Steer**: 에이전트가 작업 중일 때 새 메시지를 큐에 넣고, Steer로 진행 중인 작업을 중단하고 큐에 있는 메시지를 앞으로 이동시킬 수 있다.

**MCP 브리지**: 각 에이전트 스트림이 세션 스코프의 MCP 브리지를 실행한다. `im_list_files`(세션 워크스페이스 파일 목록), `im_send_file`(파일을 메신저에 실시간 전송) 도구가 자동 주입된다.

**GUI 자동화**: 선택적으로 브라우저 자동화(Playwright/Chrome) 및 macOS 데스크탑 자동화(Appium Mac2)를 활성화할 수 있다.

## 웹 대시보드: 에이전트 콘솔

pikiclaw v0.3+의 가장 인상적인 기능 중 하나. `http://localhost:3939`에서 IM 없이도 에이전트를 직접 제어할 수 있는 완전한 웹 인터페이스다.

- **대화 기록**: 도구 사용 활동, thinking 트레이스 포함 전체 기록
- **플랜 추적**: 에이전트의 계획 진행 상황 실시간 확인
- **스트리밍 출력**: 토큰이 생성되는 것을 실시간으로 보는 스트리밍 뷰
- **이미지 첨부**: 이미지를 에이전트에게 직접 전달
- **입력 작성기**: 멀티라인 입력, 파일 첨부 지원

IM 앱을 설정하기 전에도, 또는 IM과 병행해서 사용할 수 있다.

## 스킬 시스템

pikiclaw는 프로젝트별 스킬 파일을 지원한다. 위치 우선순위:

1. `.pikiclaw/skills/*/SKILL.md` (프로젝트 전용)
2. `.claude/commands/*.md` (Claude Code 호환)
3. `.claude/skills` / `.agents/skills` (레거시 지원, 마이그레이션 경로 포함)

메신저에서 `/skills`로 목록 확인, `/sk_<name>`으로 실행. 프로젝트에 맞는 반복 작업을 스킬화하면 한 명령으로 복잡한 워크플로우 시작.

## GUI 자동화 기능

선택적으로 두 가지 GUI 자동화를 활성화할 수 있다.

### 브라우저 자동화

`@playwright/mcp`를 통한 관리되는 Chrome 프로필. 한 번 로그인하면 모든 세션에서 재사용된다.

```
Telegram에서: "Chrome을 열고 GitHub에서 내 오픈 PR을 확인해줘"
→ pikiclaw → Claude Code → Playwright MCP → 브라우저 자동화
→ PR 목록과 상태를 Telegram으로 전송
```

### macOS 데스크탑 자동화

Appium Mac2를 통한 데스크탑 자동화:
- `desktop_open_app` — 앱 실행
- `desktop_snapshot` — 화면 상태 스냅샷
- `desktop_click`, `desktop_type` — 클릭 및 타이핑
- `desktop_screenshot` — 스크린샷 촬영

스마트폰에서 "XCode 시뮬레이터 열고 내 앱의 로그인 화면 스크린샷 찍어줘"라는 명령이 가능해진다.

## 기술 스택과 아키텍처

- **언어**: TypeScript 95.5%, JavaScript 2.4%, HTML 1.2%
- **프레임워크**: Node.js 18+
- **상태 저장**: `~/.pikiclaw/setting.json`
- **GUI 자동화**: Playwright + Appium Mac2
- **MCP 통합**: 세션 스코프 MCP 브리지 (im_list_files, im_send_file)
- **아키텍처 문서**: `ARCHITECTURE.md`, `INTEGRATION.md`, `TESTING.md`

## 설치와 사용법

```bash
# 실행 (자동으로 최신 버전 다운로드)
cd your-workspace
npx pikiclaw@latest
```

브라우저에서 `http://localhost:3939` 열기 → IM 채널 연결 → 에이전트 설정.

**환경 확인**:
```bash
npx pikiclaw@latest --doctor
```

**개발용 실행**:
```bash
git clone https://github.com/xiaotonng/pikiclaw.git
cd pikiclaw
npm install
npm run build
npm run dev    # 로컬 개발 서버, 로그 → ~/.pikiclaw/dev/dev.log
```

**IM 명령어**:
```
/start     현재 에이전트, 작업 디렉토리 정보
/sessions  세션 조회/전환/생성
/agents    에이전트 전환 (Claude Code ↔ Codex ↔ Gemini)
/models    현재 모델 확인, reasoning effort 변경
/switch    작업 디렉토리 탐색/전환
/status    런타임 상태, 토큰, 사용량
/skills    프로젝트 스킬 목록
/sk_<name> 스킬 실행
```

일반 텍스트는 현재 에이전트에게 직접 전달된다.

## 마치며

pikiclaw는 "AI 에이전트를 항상 내 손 안에"라는 비전을 실체화한 프로젝트다. 공식 CLI를 감싸서 대체하는 것이 아니라 브릿지 역할만 하므로, Claude Code나 Codex의 최신 기능과 성능이 그대로 유지된다.

v0.2에서 86번 릴리즈를 반복하며 v0.3.14까지 빠르게 발전 중이다. 로드맵에는 ACP(Agent Client Protocol) 채택, 더 많은 IM 채널(WhatsApp 등), 세션 스코프 MCP 브리지 확장이 예정되어 있다.

자리를 떠나도, 스마트폰만 있으면 AI 에이전트가 내 컴퓨터에서 계속 일하게 하고 싶다면. pikiclaw가 그 다리다.

## 소개

SSHPilot은 Linux/macOS 환경에서 SSH 접속을 체계적으로 관리하기 위한 오픈소스 도구입니다. README 기준으로 단순 터미널 래퍼가 아니라, 서버 그룹 관리·SFTP 파일 작업·포트 포워딩까지 포함한 통합 워크플로우를 제공합니다.

Termius, PuTTY, MobaXterm 대안을 찾는 사용자에게 특히 유용하며, 보안 측면에서도 libsecret 기반 자격 증명 저장을 강조합니다.

## 주요 기능

- **탭 기반 SSH 세션**으로 여러 서버를 동시 관리
- **SFTP 파일 매니저**로 원격 파일 작업 단순화
- **Local/Remote/Dynamic 포트포워딩** 지원
- **SCP 업/다운로드** 및 `ssh-copy-id` 지원
- **개인정보 표시 토글**(호스트/IP 숨김)과 테마/폰트 커스터마이징

### 핵심 포인트 1 — 운영자 친화적 UI

단순 CLI 중심 사용에서 발생하는 맥락 전환 비용을 줄여주는 것이 강점입니다. 서버를 그룹화하고 탭으로 정리할 수 있어 다중 인프라 운영 시 실수를 줄이는 데 도움이 됩니다.

### 핵심 포인트 2 — 보안·실무 기능의 균형

비밀번호/패스프레이즈를 평문으로 다루지 않는 설계와 함께, 포트포워딩·SCP·로그인 후 명령 실행 같은 실무 기능이 균형 있게 들어가 있습니다.

## 설치 / 사용법

```bash
# Debian/Ubuntu APT
sudo apt update
sudo apt install sshpilot

# 소스 실행
python3 run.py
python3 run.py --verbose
```

또는 Flatpak(`io.github.mfat.sshpilot`), AUR 패키지도 제공됩니다.

## 활용 사례 / 사용 시나리오

1. **DevOps/SRE**: 여러 환경의 SSH 접근을 탭으로 관리
2. **백엔드 운영**: 포트포워딩+파일전송을 하나의 앱에서 처리
3. **보안 민감 환경**: 비밀정보 처리 정책을 지키며 원격 접속 수행

## 결론

SSHPilot은 “가벼우면서도 실무 기능이 충분한 SSH 관리자”라는 포지션이 분명합니다. 터미널 위주의 운영 습관을 유지하면서 생산성을 높이고 싶은 사용자에게 특히 적합합니다.

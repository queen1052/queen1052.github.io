## 소개

Hypatia는 AI 에이전트를 위한 구조적 지식 관리 시스템입니다. "알렉산드리아 도서관의 두루마리들을 상상하며 거닐 수 있다. 그 파괴는 경고다: 우리가 가진 모든 것은 일시적이다." — 알베르토 망구엘의 이 말이 프로젝트의 철학을 담습니다.

지식 항목(노드)과 스테이트먼트 트리플(엣지)로 이루어진 그래프로 구조화된 지식을 저장하며, 커스텀 JSON 검색 표현(JSE) 언어로 쿼리합니다. SQLite FTS5와 DuckDB 위에 구축되어 외부 임베딩 모델 의존성이 전혀 없습니다.

GitHub에서 131 스타를 획득한 이 프로젝트는 Rust 95.2%, Shell 4.8%로 구성되어 있으며, 단 한 명의 기여자(MarchLiu/Mars Liu)가 개발하고 있습니다. MemPalace의 방법론을 적용한 벤치마크에서 ChromaDB 벡터 기반 대비 더 높은 Recall과 10-100배 낮은 레이턴시를 달성했습니다.

## 주요 기능

**지식 그래프:** 태그가 달린 이름 있는 정보 지점인 Knowledge 항목과, 시간 범위가 있는 주어-술어-목적어 Statement 트리플을 저장합니다. 각 항목에는 동의어 목록을 추가할 수 있어 도메인 특화 용어를 FTS에서 인덱싱합니다.

**JSE 쿼리 엔진:** JSON 기반 쿼리 언어가 매개변수화된 SQL + FTS5로 컴파일됩니다. `$and`, `$or`, `$not`, `$eq`, `$ne`, `$gt`, `$lt`, `$contains`, `$like`, `$content`, `$search`, `$quote`, `$triple` 연산자를 지원합니다.

**이중 데이터베이스 저장:** DuckDB는 구조적 쿼리를, SQLite FTS5(Porter 스테머 + 다중 컬럼 BM25)는 전문 검색을 담당합니다. 두 데이터베이스는 자동으로 동기화됩니다.

**Shelf 시스템:** 격리, 연결, 내보내기가 가능한 이름 있는 데이터 디렉토리로 프로젝트별 지식을 분리 관리합니다.

**에이전트 통합:** 자연어를 CLI 명령으로 변환하는 Claude Code 스킬이 포함되어 있습니다.

## JSE 쿼리 언어 심층 분석

JSE(JSON Search Expression)는 Hypatia의 핵심 차별점입니다. 구조적 쿼리와 전문 검색을 하나의 통일된 인터페이스로 제공합니다.

```bash
# 기본 지식 CRUD
hypatia knowledge-create "Rust" -d "시스템 프로그래밍 언어" -t "language,compiled"
hypatia statement-create "Rust" "is_a" "systems language"

# 전문 검색
hypatia search "programming language"

# 구조적 JSE 쿼리
hypatia query '["$knowledge", ["$eq", "name", "Rust"]]'
hypatia query '["$statement", ["$triple", "Rust", "$*", "$*"]]'

# 복합 쿼리: 태그와 이름 조합
hypatia query '["$knowledge", ["$and", ["$eq", "name", "Rust"], ["$contains", "tags", "systems"]]]'

# 트리플 매칭: Alice의 모든 관계
hypatia query '["$statement", ["$triple", "Alice", "$*", "$*"]]'

# FTS 전문 검색
hypatia query '["$knowledge", ["$search", "database migration"]]'

# 콘텐츠 필터링
hypatia query '["$knowledge", ["$content", {"format": "markdown"}]]'
```

JSE 연산자 목록:
- 비교: `$eq`, `$ne`, `$gt`, `$lt`, `$gte`, `$lte`
- 패턴: `$like` (SQL LIKE), `$contains` (JSON 필드 내 부분 문자열)
- 전문 검색: `$search` (FTS5), `$content` (컨텐츠 JSON 키-값 매칭)
- 논리: `$and`, `$or`, `$not`
- 특수: `$quote` (평가 방지), `$triple` (트리플 위치 매칭)

## 성능 벤치마크

MemPalace의 방법론(심어진 알려진-답 엔트리를 가진 합성 데이터)을 적용한 벤치마크 결과:

**소규모 테스트 (1K 지식, 2K 스테이트먼트, Apple Silicon, 디버그 빌드):**
- Recall@1: **100.0%** (20/20 needles)
- Recall@5: **100.0%**
- Recall@10: **100.0%**
- FTS 검색 p50: **474 μs**
- FTS 검색 p99: **700 μs**
- JSE 쿼리 p50: **3.39 ms**
- 인제스트 처리량: 384 knowledge/s, 280 statements/s

**MemPalace(ChromaDB 벡터 기준선) 대비:**
| 지표 | Hypatia | MemPalace |
|------|---------|-----------|
| Recall@1 | 100.0% | — |
| Recall@5 | 100.0% | 96.6% |
| 검색 레이턴시 p50 | 474 μs | ~2-50 ms |
| 임베딩 모델 | 없음 | bge-large/OpenAI |
| 콜드 스타트 | 없음 | 수 초 (모델 로딩) |
| 결정론적 | Yes | No (확률적) |

Porter 스테머 + 다중 컬럼 BM25 가중치로 단어 형태 변형을 처리하고, synonyms 필드로 도메인 특화 용어를 커버합니다. 단, 합성 데이터 기반 결과로 실제 환경에서 다를 수 있음을 개발자 자신이 솔직하게 인정하고 있습니다.

## 기술 스택 및 아키텍처

```
src/
├── cli/            # CLI 명령어 + REPL (clap + rustyline)
├── engine/         # JSE 파서, AST, 평가기, SQL 빌더
├── model/          # Knowledge, Statement, Content, Query 타입
├── service/        # 비즈니스 로직 (DuckDB + SQLite 이중 쓰기)
├── storage/        # DuckDB 저장소, SQLite FTS5 저장소, shelf 관리자
├── lab.rs          # 최상위 API 파사드
└── main.rs         # 진입점
```

각 Shelf는 `data.duckdb`(구조적 데이터)와 `index.sqlite`(FTS5 인덱스)를 포함하는 디렉토리입니다. 서비스 레이어가 이중 쓰기로 두 데이터베이스를 동기화합니다.

**18+ 플랫폼 크로스 컴파일:** x86_64/aarch64/armv7 Linux(glibc + musl), riscv64, s390x, powerpc64le, macOS, Windows, FreeBSD, NetBSD, illumos, Android를 지원합니다.

## 설치 및 사용법

```bash
# 빌드
cargo build --release

# 지식 생성
hypatia knowledge-create "Rust" -d "systems programming language" -t "language,compiled"

# 관계 생성
hypatia statement-create "Rust" "is_a" "systems language"

# 전문 검색
hypatia search "programming language"

# 대화형 REPL
hypatia repl

# Shelf 연결
hypatia connect /path/to/shelf -n myproject

# Shelf 내보내기
hypatia export myproject /path/to/export

# 벤치마크 실행
cargo test --test bench
BENCH_SCALE=medium cargo test --test bench --release

# Linux musl 정적 바이너리 빌드
./scripts/build.sh x86_64-unknown-linux-musl

# 모든 18개 타깃 빌드
./scripts/build.sh all
```

## 마치며

Hypatia는 벡터 데이터베이스의 확률적이고 느린 접근 방식 대신, 결정론적이고 초고속인 BM25 기반 전문 검색으로 AI 에이전트 메모리를 구현합니다.

임베딩 모델이 전혀 필요 없으면서도 더 높은 Recall과 10-100배 낮은 레이턴시를 달성한 결과가 인상적입니다. Rust의 성능과 안전성, DuckDB와 SQLite의 검증된 저장 엔진, 그리고 커스텀 JSE 쿼리 언어의 조합은 AI 에이전트의 장기 메모리 관리에 새로운 접근법을 제시합니다. 알렉산드리아 도서관의 지식이 불에 탔던 것처럼, AI 에이전트의 맥락이 세션마다 사라지는 문제를 해결하려는 철학이 담긴 프로젝트입니다.

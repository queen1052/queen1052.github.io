## 소개

Flowise: 드래그앤드롭 LLM 워크플로우 빌더는 에이전트와 RAG 흐름을 시각적으로 구성할 수 있는 노코드 도구입니다.라는 설명이 잘 어울리는 프로젝트입니다. 이 저장소가 흥미로운 이유는 단순히 기능 목록을 늘어놓는 데서 끝나지 않고, LLM 워크플로우를 시각적으로 설계하고 배포하는 것 같은 실제 문제를 비교적 분명한 방식으로 다루기 때문입니다.

README를 기준으로 보면 이 프로젝트는 LLM 워크플로우를 시각적으로 설계하고 배포하는 것라는 목표를 위해 필요한 구성 요소를 직접적으로 드러냅니다. 사용자는 복잡한 시행착오 없이 빠르게 결과를 확인할 수 있고, 팀 입장에서는 반복 가능한 흐름을 바탕으로 도입 여부를 판단하기 쉽습니다. 오픈소스 프로젝트를 고를 때 중요한 것은 화려한 데모보다도 “내 작업에 바로 연결할 수 있는가”인데, 이 저장소는 그 점에서 꽤 선명한 인상을 줍니다.

## 주요 기능

- 노드 기반 UI로 프롬프트, 체인, 메모리, 툴 호출 흐름을 시각적으로 조합할 수 있습니다.
- 여러 LLM provider와 벡터 스토어, RAG 구성요소를 쉽게 연결합니다.
- 프로토타입 데모부터 내부 툴 제작까지 low-code로 빠르게 전개할 수 있습니다.
- 비개발자와 개발자가 함께 흐름을 논의하기 좋은 시각적 표현을 제공합니다.
- 에이전트 오케스트레이션을 화면에서 설명하고 검증하기 쉬운 점이 강점입니다.

### 핵심 포인트 1

이 프로젝트의 첫 번째 강점은 노드 기반 UI로 프롬프트, 체인, 메모리, 툴 호출 흐름을 시각적으로 조합할 수 있습니다.라는 점입니다. 단순한 샘플 수준이 아니라 실제 문제를 어떤 데이터 흐름과 도구 체계로 풀어낼지 보여 주기 때문에, 처음 접하는 사람도 도입 방향을 잡기 쉽습니다. 또한 여러 LLM provider와 벡터 스토어, RAG 구성요소를 쉽게 연결합니다. 같은 요소는 실사용 단계에서의 마찰을 줄여 주며, 기존 작업 환경과 자연스럽게 연결될 가능성을 높여 줍니다.

### 핵심 포인트 2

아키텍처 관점에서도 노드 기반 flow builder와 다양한 모델·도구 통합을 결합한 구조라는 점이 분명합니다. 이런 구조는 유지보수성과 확장성에 직접 영향을 주며, 향후 새로운 모델·도구·워크플로우를 붙일 때도 부담을 줄여 줍니다. 특히 코드보다 빠르게 LLM 파이프라인을 실험할 수 있다는 점은 비슷한 오픈소스와 비교할 때 이 프로젝트를 다시 보게 만드는 차별점입니다.

## 설치 / 사용법

아래는 README 기준으로 시작할 때 참고하기 좋은 대표 명령들입니다.

```bash
git clone https://github.com/FlowiseAI/Flowise
cd Flowise
npm install -g flowise
npx flowise start
```

설치 후에는 예제 시나리오나 기본 워크플로우부터 먼저 실행해 보는 것이 좋습니다. 환경 변수, API 키, 외부 서비스 연결이 필요한 프로젝트라면 README의 설정 섹션을 함께 확인해야 실제 동작을 더 빠르게 재현할 수 있습니다.

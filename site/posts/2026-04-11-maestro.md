## 소개: "내 머신에서 실행, 공유 UI로 접근"

AI 에이전트 오케스트레이션 도구는 보통 중앙 서버에 모든 데이터를 올린다. Maestro는 다르다. 각 사용자가 **자신의 머신에서 서버를 실행**하고, 공개 공유 프론트엔드에 연결하는 방식이다.

**Maestro**는 olsommer가 개발한 서버 기반 AI 에이전트 오케스트레이션 플랫폼이다. GitHub 8 stars, TypeScript 92.9%, v0.1.63, 67개 릴리즈를 기록한 빠르게 성장 중인 프로젝트다. Kanban 보드로 에이전트 작업을 시각화하고, 슈퍼 에이전트와 자동화 엔진으로 여러 AI 코딩 에이전트를 조율한다.

## 주요 기능 및 아키텍처

**공유 프론트엔드 + 개인 백엔드** 아키텍처가 핵심이다:

```
공유 프론트엔드 (Next.js)
https://maestro-beige.vercel.app
        │
   User A URL + API token    User B URL + API token
        │                           │
        ▼                           ▼
Tailscale (User A)          Tailscale (User B)
HTTPS + MagicDNS            HTTPS + MagicDNS
        │                           │
        ▼                           ▼
Server (User A)             Server (User B)
localhost:4800              localhost:4800
```

각 사용자는 자신의 머신에서 서버를 실행하고, Tailscale로 안전하게 외부 접근을 허용한다. 프론트엔드는 공유되지만 데이터는 각자의 서버에 격리된다.

**보안 모델**:
- 프론트엔드에 비밀이 없음 — 공개 호스팅이 안전함
- 각 서버는 자신의 API 토큰만 수락 — 사용자 간 상호 접근 불가
- 모든 트래픽은 Tailscale 터널(HTTPS) 통과 — 서버가 TLS를 직접 처리하지 않음
- API 토큰은 `~/.maestro/token`에 권한 `0600`으로 저장

**Node.js `node:sqlite` 모듈** 기반으로 외부 데이터베이스 없이 SQLite를 사용한다. Node.js 22.13.0+ 필수 조건이다.

## 빠른 시작

**글로벌 CLI 설치** (가장 빠른 방법):

```bash
# Node.js 22.13.0+ 확인
nvm install 22
nvm use 22

# Maestro CLI 설치
npm i -g @isarai/maestro

# 온보딩 (최초 1회)
maestro onboard

# 서버 시작
maestro start

# 상태 확인
maestro status
```

첫 실행 시 API 토큰이 `~/.maestro/token`에 자동 생성된다.

**Tailscale로 외부 접근 허용**:

```bash
# macOS
brew install tailscale

# Tailscale 인증
sudo tailscale up

# 백엔드를 HTTPS로 서빙 (백그라운드)
tailscale serve --bg http://localhost:4800

# MagicDNS 호스트명 확인
tailscale status --json | grep MagicDNSSuffix
```

이후 `https://<machine-name>.<tailnet>.ts.net`이 서버 URL이 된다. 자동 TLS가 적용되어 포트 번호 없이 443 → localhost:4800으로 프록시된다.

**공유 프론트엔드 연결**: `https://maestro-beige.vercel.app`에 접속 → 서버 URL + `~/.maestro/token` 내용 입력 → 연결.

## CLI 명령어 및 환경 변수

**유용한 CLI 명령어**:

```bash
maestro logs          # 로그 출력
maestro logs -f       # 실시간 로그 스트리밍
maestro auth          # 인증 관리
maestro version       # 버전 확인
maestro update --check  # 업데이트 확인 (자동 적용 안 함)
maestro stop          # 서버 중지
```

**환경 변수**:

| 변수 | 기본값 | 설명 |
|------|-------|------|
| PORT | 4800 | 서버 수신 포트 |
| HOST | 0.0.0.0 | 서버 바인딩 주소 |
| AUTH_DISABLED | — | 1로 설정 시 인증 비활성화 (개발 전용) |
| GITHUB_TOKEN | — | GitHub 통합용 토큰 |

## 로컬 개발 및 프로젝트 구조

**로컬 개발 환경**:

```bash
# 의존성 설치
pnpm install

# 서버 시작 (포트 4800)
pnpm dev:server

# 클라이언트 시작 (포트 3000)
pnpm dev:client

# 브라우저에서 http://localhost:3000 접속
# 서버 URL: http://localhost:4800
# 토큰: ~/.maestro/token 내용 붙여넣기
```

**프로젝트 구조**:

```
maestro/
├── packages/
│   ├── server/   # Fastify API + Socket.io + PTY + agents
│   ├── client/   # Next.js 웹 인터페이스
│   ├── wire/     # 공유 Zod 스키마 & 타입
│   └── mcp/      # 슈퍼 에이전트용 MCP 오케스트레이터
└── pnpm-workspace.yaml
```

**서버**는 Fastify API + Socket.io 실시간 통신 + PTY(가상 터미널) + 에이전트 관리로 구성된다. **클라이언트**는 Next.js 기반 Kanban 보드 웹 UI다. **wire** 패키지가 Zod 스키마로 타입 안전성을 보장한다. **mcp** 패키지가 슈퍼 에이전트의 MCP 오케스트레이션을 담당한다.

**상태 관리**: Maestro 소유 상태는 `~/.maestro/`에, 프로젝트는 `~/.maestro/projects/`에, 샌드박스는 `~/.maestro/sandboxes/`에 저장된다. 에이전트 Docker 샌드박싱을 원한다면 Docker가 필요하다.

## 프론트엔드 셀프호스팅 (선택)

대부분의 사용자는 공유 프론트엔드를 사용하면 된다. 커스텀 호스트나 브랜딩이 필요하면 직접 배포할 수 있다:

```bash
cd packages/client
vercel deploy --prod
```

프론트엔드에는 비밀이 없어 공개적으로 안전하게 호스팅할 수 있다. Connect 페이지에서 각 사용자가 자신의 백엔드 URL을 입력하는 방식이다.

**바어메탈 서버 배포**:

```bash
# Node.js 22.13.0+ 필요
pnpm install
NODE_ENV=production pnpm dev:server
# 첫 실행 시 토큰이 출력되고 ~/.maestro/token에 저장됨
```

Docker는 Maestro 서버 자체에는 필요 없고, 에이전트 Docker 샌드박싱에만 필요하다.

## 마치며

Maestro는 "중앙 집중 없이, 개인의 머신에서 AI 에이전트를 운영한다"는 철학의 도구다. 공유 UI와 개인 백엔드를 분리한 아키텍처는 데이터 프라이버시와 사용 편의성을 동시에 만족시키는 영리한 설계다.

Tailscale과의 통합으로 VPN 없이도 어디서나 안전하게 자신의 에이전트 서버에 접근할 수 있다. 빠른 릴리즈 사이클(67개 릴리즈)이 보여주듯 활발히 개발 중인 프로젝트로, AI 코딩 에이전트를 체계적으로 오케스트레이션하고 싶은 개발자에게 관심을 가질 만한 도구다.
